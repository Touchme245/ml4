# Отчет по лабораторной работе
## Исследование моделей семантической сегментации дорог на спутниковых снимках

---

## 1. Выбор начальных условий

### 1.1 Выбор набора данных

В данной работе использован набор данных для сегментации дорог на спутниковых снимках (Road Segmentation Dataset). Данные представляют собой спутниковые изображения и соответствующие им бинарные маски, где выделены дорожные участки.

Структура датасета:

- training/images - 100 оригинальных спутниковых снимков
- training/groundtruth - 100 масок дорог для оригинальных снимков
- training/images_generated - 1000 сгенерированных спутниковых снимков (Pix2Pix)
- training/groundtruth_generated - 1000 масок для сгенерированных снимков
- test_set_images - 50 тестовых изображений

Обоснование выбора набора данных:

Практическая задача автоматического выделения дорог на спутниковых снимках имеет высокую актуальность для таких областей, как обновление навигационных карт, мониторинг дорожной инфраструктуры, планирование маршрутов в труднодоступных регионах, оценка последствий стихийных бедствий и автоматическое картографирование.

Данный датасет выбран по следующим причинам. 

Задача имеет реальную практическую значимость и широко востребована в геоинформационных системах. 

Структура датасета позволяет исследовать влияние сгенерированных данных на качество моделей. 

Дисбаланс классов (дороги занимают около 5-10 процентов площади изображения) создает вызов для моделей семантической сегментации.

### 1.2 Выбор метрик качества

Для оценки качества моделей семантической сегментации в данной работе использованы следующие метрики.

Dice Score вычисляется по формуле 2 * TP / (2 * TP + FP + FN). Данная метрика выбрана потому, что она оптимальна для несбалансированных данных, когда целевой класс занимает малую площадь изображения.

IoU вычисляется по формуле TP / (TP + FP + FN). Это стандартная метрика в соревнованиях по сегментации, таких как DeepGlobe и SpaceNet.

Precision вычисляется по формуле TP / (TP + FP). Эта метрика важна для минимизации ложных обнаружений дорог.

Recall вычисляется по формуле TP / (TP + FN). Эта метрика важна для обнаружения всех существующих дорог на изображении.

Комбинация этих четырех метрик позволяет получить полную картину качества сегментации, учитывая как общую точность, так и баланс между полнотой и точностью.

## 2. Создание бейзлайна и оценка качества

### 2.1 Выбор моделей

Для создания бейзлайна использованы две модели из библиотеки segmentation_models_pytorch.

Первая модель - сверточная (CNN) архитектура U-Net с энкодером ResNet34

Вторая модель - архитектура на основе трансформеров: U-Net с энкодером mit_b0 (MixTransformer)

### 2.2 Условия обучения

Обучение проводилось при следующих параметрах:

- валидационная выборка: 20 процентов от обучающей
- количество эпох: 10
- размер батча: 8
- оптимизатор: Adam
- learning rate: 3e-3
- функция потерь: Dice Loss (from_logits=True)
- размер изображения: 256x256 пикселей

### 2.3 Результаты бейзлайна

Результаты обучения моделей на исходном датасете:

Модель CNN Baseline (U-Net + ResNet34) показала следующие результаты: Dice: 0.8830 | IoU: 0.7911 | Precision: 0.8611 | Recall: 0.9066

Модель ViT Baseline (U-Net + mit_b0) показала следующие результаты: Dice: 0.8763 | IoU: 0.7807 | Precision: 0.9205 | Recall: 0.8370

## 3. Улучшение бейзлайна

### 3.1 Сформулированные гипотезы

Для улучшения качества моделей были сформулированы следующие гипотезы.

Гипотеза 1: Применение расширенных аугментаций данных (повороты, отражения, изменение яркости и контраста, гамма-коррекция) повысит устойчивость моделей к вариативности входных данных.

Гипотеза 2: Использование оптимизатора AdamW с уменьшенным learning rate обеспечит лучшую регуляризацию и более стабильную сходимость.

Гипотеза 3: Добавление механизма внимания scSE в декодер улучшит фокусировку модели на релевантных признаках.

Гипотеза 4: Использование комбинированной функции потерь Dice + BCE позволит учесть дисбаланс классов и обеспечит более стабильные градиенты.

### 3.2 Улучшенный бейзлайн

На основе проверки гипотез сформирован улучшенный бейзлайн, включающий следующие компоненты:

- аугментации: горизонтальное отражение, вертикальное отражение, случайный поворот на 90 градусов, случайное изменение яркости и контраста, случайная гамма-коррекция
- оптимизатор: AdamW с learning rate 1e-4 для CNN и 1e-3 для ViT
- функция потерь: комбинированная Dice + BCE
- для ViT дополнительно добавлен механизм внимания scSE в декодер

### 3.3 Результаты улучшенного бейзлайна

Обучение улучшенного бейзлайна дало следующие результаты.

Модель CNN Enhanced: Dice: 0.9382 | IoU: 0.8839 | Precision: 0.9334 | Recall: 0.9433

Модель ViT Enhanced: Dice: 0.6911 | IoU: 0.5376 | Precision: 0.7619 | Recall: 0.6019

### 3.4 Сравнение результатов

Сравнение улучшенного бейзлайна с исходным показало следующее.

Для модели CNN получилось добиться значительного улучшения точности

Для модели ViT наблюдается ухудшение результата, модель не дообучилась, нужно повысить количество эпох, тк прирост между эпохами очень существенный

## 4. Имплементация алгоритмов машинного обучения

### 4.1 Самостоятельная имплементация моделей

В рамках работы были самостоятельно имплементированы две модели.

Первая модель - сверточная архитектура U-Net.

Вторая модель - трансформерная архитектура SimpleViTSegmenter.

### 4.2 Обучение имплементированных моделей

Обучение собственных моделей проводилось с использованием оптимизатора Adam с learning rate 1e-3 и функцией потерь BCEWithLogitsLoss.

Собственная модель CNN показала следующие результаты: Dice: 0.8738 | IoU: 0.7764 | Precision: 0.8598 | Recall: 0.8888

Собственная модель ViT показала следующие результаты: Dice: 0.6723 | IoU: 0.5067 | Precision: 0.6601 | Recall: 0.6854


### 4.3 Добавление техник улучшенного бейзлайна

К собственным моделям были добавлены техники из улучшенного бейзлайна, а именно:

- расширенные аугментации данных
- оптимизатор AdamW с подобранными learning rate
- комбинированная функция потерь Dice + BCE

### 4.4 Обучение улучшенных собственных моделей

После добавления техник улучшенного бейзлайна собственные модели показали следующие результаты.

Модель Own CNN Enhanced: Dice: 0.8725 | IoU: 0.7748 | Precision: 0.8530 | Recall: 0.8940

Модель Own ViT Enhanced: Dice: 0.7432 | IoU:  0.5912 | Precision: 0.7312 | Recall: 0.7567
